## Codebase Patterns
- Layout `src/` com `PYTHONPATH=src` para executar sem instalar o pacote
- Dependências pesadas (torch, ultralytics, playwright) DEVEM falhar com erros acionáveis quando ausentes — use `try/except ImportError` e levante erro customizado com mensagem de install
- Tests que precisam de torch/ultralytics usam `monkeypatch.setattr` para evitar dependência real
- Formato YOLO: "class_id cx cy w h" normalizado 0-1, seis casas decimais
- Sidecar JSON alinhado linha-a-linha com o .txt YOLO (índice i do JSON = bbox i do .txt)
- TDD: sempre escreva o teste que falha ANTES de escrever a implementação
- `venv/bin/python -m pytest -v` para rodar testes
- `venv/bin/python -m ruff check .` para lint
- `venv/bin/python -m mypy src` para typecheck
- O `split_for_id` usa SHA1 do nome do arquivo módulo 10 para split determinístico
- `DatasetWriter` cria estrutura images/labels/attrs/{train,val,test}/
- Para mockar módulos opcionais em tests: use `monkeypatch.setitem(sys.modules, "modulo", mock)` — mais robusto que setattr pois intercepta o `sys.modules.get()` usado em `_load_ultralytics()`
- Sentinel pattern para detectar módulo ausente: `sentinel = object(); mod = sys.modules.get("mod", sentinel); if mod is sentinel: importlib.import_module(...)`; nunca usar `...` (Ellipsis) como sentinel com mypy pois causa conflito de tipos
- Isolamento de módulo em tests: quando um test injeta torch/ultralytics mock ANTES de importar o módulo, use `monkeypatch.delitem(sys.modules, "ia_visao_web.model.XXX", raising=False)` no helper para garantir que o módulo seja reimportado com o mock — sem isso o módulo cacheado do test anterior pode ser reutilizado
- Mypy `ignore_missing_imports = true` para torch: o `# type: ignore[assignment]` em `_mod = None` é desnecessário (a anotação seria `unused-ignore`)
- Mypy `no-redef` em try/except: se declarar `var: Any` antes do try e depois `from pkg import var`, adicionar `# type: ignore[no-redef]` na linha do import
